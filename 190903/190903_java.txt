지역변수랑 구분하려고 전역변수에는 this. 으록 구분합니다
this == 이 인스턴스에!! 메서드의 지역변수도 아니고 힙영역의 인스턴스임
this() :: 이 인스턴스에 있는 생성자! 라는 뜻임. 생성자 내에서 다른 생성자를 호출할 때 사용함.


생성자끼리 호출할수있도록 하려고 this() 를 하는거임..
얘는 무조건 생성자 내의 첫줄에서만 쓸수있음


Car c = new Car("auto");
Car를 메서드영역에 올리고 c를 지역변수로 올려
new연산자로 초기화해서 인스턴스만들어
그다음에 Car() 생성자 메서드 호출해서 콜스택에 적재해
메서드의 매개변수가 있을때에만 메서드의 지역변수로 올려
gearType = auto로 올리고 Car(String) 메서드 호출해서 적재
Car() 기본생성자 메서드 처리하고 아래거하면됨


생성자를 이용한 초기화 :: 이거를 인스턴스 내 변수 입장에서 보면 기본값으로 초기화됐다가 다른값으로 저장된것처럼 보이겠지만, 걔를 부른 main 메서드 입장에서는 얘가 인스턴스화 끝나니까 초기화됐자나? 그래서 걍 초기화된거임


	static int action;
	int action2 = 10;	
	Init(){
		action2 = 20;
	}
	
	// 매개변수가 있는 생성자. int 타입 1
	Init(int action2){
		this.action2 = action2;
	}
여기서 두번째 생성자를
Init(a){ action2 = a;} 할수있겠지
근데 이렇게하면 나중에 부를때 a가 어떤값으로 들어가는지 모르는거야
이걸 위에처럼 해줘야 헷갈리지않을수있음다


초기화블럭은 new 연산자를 마친 상태에서 생성자에 가기전에 호출됨..!
생성자는 여러개중에 하나만 들리지만 초기화블럭은 무조건 들리게되어있음
초기화블럭이 여러개다? 그럼 순서대로 들린단다
