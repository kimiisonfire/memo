우리가 쓰는 toString(), equals() 메서드는 전부 Object 클래스거임

부모클래스의 메서드를 재정의 : 오버라이딩
같은 이름의 메서드를 여러개 : 오버로딩


오버라이딩에서 예외는 부모클래스의 메서드보다 많이 선언할수없다
:: 부모클래스의 메서드에서 예외(에러비슷한거)가 두개 났다
그러면 자손클래스에서는 예외가 2개 이하로 나야하는거야
부모클래스 메서드는 덜 작성해놓은 상태일수도 있으니까 예외가 여러개 날수있지
근데 그걸 받아와서 수정했는데 예외가 더나는건 안되는거지


this 일때는 같은클래스에 완전히 같은 메서드가 없으니까 그냥 변수만 this로 부르고 메서드는 걍 호출했는데
super일때는 서로다른 클래스에 완전히 같은 메서드(선언부분이)가 있을수있어서 super.~~() 이런식으로 메서드랑 변수를 부를수있음


this()랑 super()는 같이쓸수없음 당연히 첫줄에써야하니까 둘다쓸순없쥬
그러면 this()만 써서 같은 클래스내 다른 메서드를 호출하면
그 다른메서드는 super()를 하고있으니까.. 자동으로 super를 호출하는거지


접근제어자 protected ::
같은 패키지에서 접근 가능한데 그중에서도 상속받은 애들에 대해서는 다른 패키지에 있는 애들도 부를수가 있다구..!
