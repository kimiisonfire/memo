프로세스와 쓰레드의 차이점

프로그램을 작성해서 실행시키려면 컴터안에 실행할 파일이 있어야겠지? = 기계어로 번역된
윈도우/리눅스/맥/.. 등 각 운영체제에 맞게 기계언어로 번역된 파일들..
윈도우는 대부분 .exe로 끝나는 친구들이 실행파일이야
실행이 가능한 파일을 프로그램파일 또는 프로그램 이라고 헙니다
실행가능한 명령어의 집합 = 프로그램
얘를 실행시키면 안에 저장된 명령어를 하나씩 cpu가 해석을 하면서 운영체제는 프로그램이 실행될 환경을 만들어줘. 
메모리상에 프로그램명령어를 갖다놓고 현재 실행중인 상태 = 프로세스
여러개의 프로세스를 한꺼번에 한 운영체제이서 실행되는 상태 = 멀티프로세스

현재 실행중인 프로그램 즉 프로세스는 자원(데이터, 메모리,...)과 쓰레드로 구성

우리는 아직 쓰레드를 안쓴것같지만 쓰고있었다.. 
main 메서드를 실행시키는 main 쓰레드를 쓰고있다구...!

멀티프로세스 : 하나의 프로그램을 동시에 여러개 실행하는겨
멀티쓰레드 : 어떤 작업을 동시에 처리

쓰레드는 두개의 작업이 있으면 하나한테 시간을 쪼끔주고 그다음에 다른애한테 시간을 쪼금주고 이렇게 짧은 시간들을  주고주고주고 하는거임
만약에 동시에 a, b가 돈을 출금한다고하면 한줄한줄 실행하니까 둘다 잔액이 같은상태일때 출금해버리면 둘다 돈을 뺴가는거자녀 그럼 은행손해아녀 -> 동기화문제
그래서 쓰레드가 한쪽에서 돌아가는 동안 다른쪽엔 보호망같은게 쳐진대..

운영체제 내에 있는 쓰레드 스케줄러가 시간 관리를 해줌
쓰레드가 어떤때는 한쪽이 더 길고 오래해야할 그런 경우도 있지않겠어?
쓰레드 쓰면 프로그래머가 고려해야할 사항이 많아집니다..




자바에서 쓰레드를 만들어 실행하는 방법 2개가 있다
Thread 클래스 상속받기 / Runnable 인터페이스 구현하기
Runnable을 구현하면 runnable 객체 생성하고, thread 객체도만들고나서 실행..
thread 상속받을때는 좀더 간단하다네요,, 한번만 하면 된대

자바는 단일상속만 가능해서 일반적으로는 thread를 상속받아쓰고,
다중상속이 필요한 경우(이미thread 상속받았는데 thread 기능을 또 넣고싶을떄) runnable 구현해서 하는것입니다..


thread를 상속받는 클래스의 객체를 만들어서 객체참조변수명.start() 하면 돌아갑니듀
그러면 같은 call stack안에서 돌아가는게 아니고 하나의 thread 영역을 더 만드는거임
콜스택에 스택이 두개 생기는거임.. 무튼 새로만들어진 영역에 호출한 메서드를 돌려놓고 호출했던 메서드는 끝나버림

프로세스가 하나라도 작동중이면 당연히 아직 작동중인거암



자바에서 어케 쓰레드 쓰나요??
1. Thread 클래스를 상속받는 클래스 안에 run() 오버라이드하고, 객체생성해서 start()
2. Runnable 인터페이스를 구현한 구현체에서 run() 오버라이드하고
객체생성해서 그 객체를 Thread 인스턴스 생성할때 인자로 넣어줌
Thread의 인스턴스.start()
3. 클래스에 Thread 인스턴스 만드는데 생성자부분에 익명클래스 만들어서 run() 오버라이드해주고 바로 호출!


thread.start()
thread.sleep()	:: 잠시 중지
thread.join()	:: 쓰레드가 끝날때까지 대기
