아직 TO_CHAR 하는중
9  :  유효한 숫자는 데이터 자신을 출력하고 무효의 0은 공백처리. 9-MODE
---> SELECT TO_CHAR(PROD_PRICE, '9,999,999') FROM PROD;
290,000	1,780,000	  42,000
0  :  유효한 숫자는 데이터 자신을 출력하고 무효의 0은 0으로 처리. 0-MODE
---> SELECT TO_CHAR(PROD_PRICE, '0,000,000') FROM PROD;
0,290,000  1,780,000  0,042,000

제로모드는 잘 안씀. 대신 나인모드랑 섞어서 쓰기도함
SELECT TO_CHAR(PROD_SALE, '9,999,990') FROM PROD;
이렇게하면 0일때 0으로 출력하는거임


새로운 코드를 부여할때말여
숫자로만 분류할때는 MAX(컬럼명) + 1 
숫자로만 되어있으면 쉽게할수있겠지만, 문자가 껴있단말여..! 
아래처럼 해보면 됩니듀
SELECT 'P'||TO_CHAR(MAX(SUBSTR(LPROD_GU,2))+1)
  FROM LPROD
 WHERE LPROD_GU LIKE 'P4%'

TO_CHAR의 화폐 L 을 쓸때는 제일 먼저 도구-환경설정-데이터베이스-NLS-통화에 정의된 기호를 참조함. 그 다음에는 제어판-국가및언어 에 정의된 기호를 참고합니듀
--> SELECT TO_CHAR(PROD_COST,'L9,999,990') FROM PROD;
